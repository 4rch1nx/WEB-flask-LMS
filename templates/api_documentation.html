1. Зачем нужен API?

API (Application Programming Interface) - это набор правил и протоколов, которым соглашаются следовать две отдельные системы или программы, позволяющие приложениям взаимодействовать друг с другом. По сути API — это посредник, который позволяет одной программе «общаться» с другой, обмениваться нужной информацией и отображать её для пользователей. Таким образом, WEBRobotics API поможет разработчикам использовать функции нашего приложения в своих проектах!
Функции, которые поддерживает наш API:
‣ Работа с системой вопросов и ответов.
Вы можете получить список ваших вопросов, данные о конкретном вопросе, изменить и задать вопрос с помощью HTTP-запросов, а также просматривайте ответы админов на ваши вопросы!
‣ Работа с сохранёнными алгоритмами и устройствами.
Получайте подробную информацию о сохранённых вами устройствах и алгоритмах движения машинки!
‣ Подключение машинки и управление.
С помощью HTTP-запросов управляйте нашей машикной! Вы можете подключиться к машинке, отправлять запросы для отдельного движения каждого мотора и двигаться по алгоритму!
‣ Получение значений с датчиков.
На нашей машинке установленны датчики, и с помощью API вы можете получить значения с них!

Вы не можете получать с помощью API такие данные, как логины и пароли, так как при передаче данные могут быть перехвачены!





2. Ключ API

Для использования нашего API вам необходим уникальный ключ. Такой ключ выдается при регистрации в нашем приложении. Посмотреть и скопировать ключ вы можете в вашем личном кабинете в строке "Ключ API"
Пример API-ключа:

4le58yCO27UpuE0CuK08hU2P


Никому не передавайте ваш ключ API, так как с его помощью можно получить доступ к вашим личным данным!






3. Структура запроса

Для использования API используются HTTP-запросы (GET, POST, PUT, DELETE) - подробнее про них можно прочитать тут.

Здесь мы покажем, как использовать API с помощью Python, а точнее python-библиотеки requests. Однако вы можете использовать API и с помощью обычных HTTP-запросов.

Итак, общая структура запроса:



http://127.0.0.1:8080/api/название ресурса?api_key=Ваш ключ API&Другие аргументы запроса


Если же использовать Python:


Python

import requests

params = {'api_key': 'Ваш ключ API', Другие аргументы запроса }
answer = requests.get("http://127.0.0.1:8080/api/название ресурса", json=params).json()
print(answer)


Ресурсы - это единицы информации в API (например, вопросы, сохранённые устройства и т.д.)

Другие аргументы запроса (кроме ключа API, который нужен всегда) зависят от ресурса, к которому обращается пользователь. Прочитать про них можно далее.







4. Вопросы и ответы

Начнём с API для получения и работы с вопросами пользователей и ответами разработчиков.

Аргументы запроса для вопросов пользователей:


Аргумент Расшифровка
api_key Ваш ключ API
question_id ID запрашиваемого пользователем вопроса. Обязателен при GET-запросе (при получении одного элемента).
theme Тема вопроса. Необходима при добавлении вопроса с помощью POST-запроса.
question Сам вопрос, добавляемый с помощью API. Необходим при использовании POST-запроса.
